## 1. Project Overview

**App Name:** BetterAlt Fat Burner
**Package:** `fat_burner`
**Bundle ID (Android):** `com.fatburner.fat_burner`
**Firebase Project:** `fatburner---app`
**Android App ID:** `1:231893834616:android:e8e67e487c21caf4d2868d`

The app is a Flutter health tracking companion for customers who purchase the "BetterAlt Fat Burner" product sold via Shopify. The core feature is verifying that a user has purchased "Plant Protein" from Shopify and unlocking the full app experience for verified buyers.

**Core Value Propositions:**
- Daily fat burner supplement tracking (did user take it today?)
- Health metrics dashboard: steps, calories burned, sleep, weight
- Shopify purchase verification gate (Plant Protein buyer = full access)
- Firebase push notifications for reminders/engagement
- HealthKit (iOS) / Health Connect (Android) data integration

## 3. Entry Point ‚Äî main.dart

**File:** `lib/main.dart`

### What it does:
1. Calls `WidgetsFlutterBinding.ensureInitialized()`
2. Initializes Firebase with `DefaultFirebaseOptions.currentPlatform`
3. Registers the FCM background message handler (must be top-level)
4. Initializes `FirebaseMessagingService` (permissions, token, listeners)
5. Runs `FatBurnerApp`

### FatBurnerApp Widget:
- Wraps with `MultiProvider` ‚Äî currently exposes `PurchaseStatusProvider.instance`
- Uses `MaterialApp` with `AppTheme.light` / `AppTheme.dark`
- Uses `ThemeMode.system` (respects device dark/light setting)
- Home is `AuthGate` (NOT using GoRouter ‚Äî this is a known inconsistency)

### AuthGate:
- `StreamBuilder<User?>` on `FirebaseAuth.instance.authStateChanges()`
- Shows `CircularProgressIndicator` while loading
- Routes to `MainScreen` if logged in, `LoginScreen` if not

### Known Issue:
`AppRouter` (GoRouter) is defined in `core/app_router.dart` but **not connected** to `MaterialApp`. The app uses `AuthGate` for auth routing and `MainScreen`'s `BottomNavigationBar` for in-app navigation. `GoRouter` is currently dead code.

## 4. Core Layer

### app_router.dart
- Defines `AppRouter.login = '/login'` and `AppRouter.dashboard = '/dashboard'`
- GoRouter with `refreshListenable: AuthService.instance.authState`
- Redirect logic: logged in ‚Üí dashboard, not logged in ‚Üí login
- **Currently unused** ‚Äî `MaterialApp` in `main.dart` uses `home: AuthGate()` not `router:`
- `LoginScreen` calls `context.go(AppRouter.dashboard)` but this only works if GoRouter is active

### app_theme.dart
- Material3 (`useMaterial3: true`)
- Light theme: seed color `#2E7D32` (dark green), primary `#2E7D32`
- Dark theme: seed color `#66BB6A` (light green)
- `AppBarTheme`: centered title, no elevation
- `InputDecorationTheme`: rounded border (12px), filled

### firebase_messaging_service.dart
- Singleton: `FirebaseMessagingService.instance`
- `initialize()`: requests alert/badge/sound permissions, sets up foreground/background/tap handlers, gets FCM token
- iOS-specific: waits for APNs token before fetching FCM token
- `_handleForegroundMessage` and `_handleNotificationTap` are **stubs** (not implemented)
- FCM token is retrieved but **not sent to any backend** (TODO)

## 6. Services

### AuthService (`lib/services/auth_service.dart`)
- Singleton: `AuthService.instance`
- Listens to `FirebaseAuth.authStateChanges()` internally
- `authState: ValueNotifier<bool>` ‚Äî used by GoRouter's `refreshListenable`
- `currentUser: UserModel?` ‚Äî cached in memory
- `isLoggedIn: bool` ‚Äî derived from `currentUser != null`

**Key methods:**
| Method | Description |
|--------|-------------|
| `login(email, password)` | Firebase `signInWithEmailAndPassword`, returns `AuthResult` |
| `signUp(email, password, displayName?, phone?)` | Creates user, updates displayName, saves profile to Firestore |
| `logout()` | Firebase `signOut`, clears `_currentUser` |
| `checkSession()` | Returns `true` if `FirebaseAuth.currentUser != null` |

**Error codes handled:** `user-not-found`, `wrong-password`, `invalid-email`, `user-disabled`, `email-already-in-use`, `weak-password`, `invalid-credential`, `too-many-requests`, `operation-not-allowed`

**Firestore path:** `users/{uid}` ‚Äî stores `email`, `displayName`, `phone`, `createdAt`, `updatedAt`

### AuthResult (`lib/services/auth_result.dart`)
Sealed class pattern:
```dart
sealed class AuthResult {}
class AuthSuccess extends AuthResult {}
class AuthFailure extends AuthResult { final String message; }
```

### FirestoreService (`lib/services/firestore_service.dart`)
- Singleton: `FirestoreService.instance`
- `saveHealthData(userId, data)` ‚Äî writes to `users/{uid}/health_data/{YYYY-MM-DD}`
- Fields: `steps`, `calories`, `distance`, `date`, `updatedAt`
- Uses `SetOptions(merge: true)` ‚Äî safe to call multiple times per day

### HealthService (`lib/services/health_service.dart`)
- Singleton: `HealthService.instance`
- Platform: iOS (HealthKit) + Android (Health Connect)
- Types read: `STEPS`, `ACTIVE_ENERGY_BURNED`, `WEIGHT`
- All permissions are READ-only

**Key methods:**
| Method | Returns | Description |
|--------|---------|-------------|
| `requestAuthorization()` | `Future<bool>` | Prompts user for health permissions |
| `hasPermissions()` | `Future<bool>` | Checks if already authorized |
| `getTodaySteps()` | `Future<int>` | Steps from midnight to now |
| `getTodayCaloriesBurned()` | `Future<int>` | Active energy burned today |
| `getLatestWeight()` | `Future<double?>` | Most recent weight (past 365 days) |
| `fetchBasicHealthData()` | `Future<Map>` | Steps + calories + distance, normalized |
| `getTodayHealthData(userId)` | `Future<HealthDataModel?>` | Combined model |

**Normalization:** Prevents negative values; converts meters > 1000 to km.

**Note:** `HealthService` is **not yet connected to any screen**. All dashboard stats are hardcoded placeholder values.

### ShopifyPurchaseService (`lib/services/shopify_purchase_service.dart`)
- Singleton: `ShopifyPurchaseService.instance`
- Calls Firebase Cloud Function `checkPlantProteinPurchase` via `FirebaseFunctions.instance.httpsCallable`
- `hasPurchasedPlantProtein(email?, phone?)` ‚Äî returns `Future<bool>`
- Requires at least one of email/phone; throws `ArgumentError` otherwise

## 7. Providers (State Management)

### PurchaseStatusProvider (`lib/providers/purchase_status_provider.dart`)
- Singleton `ChangeNotifier`: `PurchaseStatusProvider.instance`
- Registered in `main.dart` via `ChangeNotifierProvider.value`
- Wraps `ShopifyPurchaseService` with loading/error state

**State:**
```
isLoading: bool     ‚Äî true while Firebase CF is being called
error: String?      ‚Äî human-readable error message
hasPurchased: bool? ‚Äî null = not checked, true/false = result
hasResult: bool     ‚Äî true when hasPurchased != null && error == null
```

**Methods:**
- `checkPurchase(email?, phone?)` ‚Äî triggers the Shopify check
- `clearError()` ‚Äî resets error state
- `reset()` ‚Äî clears all state

**Error codes handled:** `unauthenticated`, `invalid-argument`, `failed-precondition`, `internal`, `unavailable`

**Note:** `PurchaseStatusProvider` is registered but **never called from any screen**. The purchase gate UI is not yet built.

## 8. Screens

### LoginScreen (`lib/screens/login_screen.dart`)
- Handles both login and sign-up (toggle via `_isLoginMode`)
- Uses `GlobalKey<FormState>` for validation
- Fields: Display Name (signup only), Email, Phone (signup only), Password
- Email validation: non-empty + contains `@`
- Password validation: non-empty + min 6 chars
- Error display: inline container with error icon + message
- On success: calls `context.go(AppRouter.dashboard)` (GoRouter ‚Äî currently broken since GoRouter not active)
- Loading state: disables button, shows `CircularProgressIndicator`

### MainScreen (`lib/screens/main_screen.dart`)
- Bottom navigation shell with 4 tabs
- Tabs: Home (index 0), Steps (index 1), Calories (index 2), Profile (index 3)
- Uses `BottomNavigationBar` with green selected color

### HomeScreen (`lib/screens/home_screen.dart`)
- Loads today's `fatBurnerTaken` status from Firestore on `initState`
- Firestore path: `users/{uid}/daily/{YYYY-MM-DD}`
- `Switch` widget toggles `fatBurnerTaken` and writes back to Firestore
- Stats grid: 4 cards (Steps, Calories, Sleep, Weight) with `CircularProgressIndicator` rings
- **All stat values are hardcoded** (1240 steps, 320 cal, 7.5h sleep, 72kg)
- Does not yet call `HealthService`

### DashboardScreen (`lib/screens/dashboard_screen.dart`)
- Separate route `/dashboard` (via GoRouter)
- Visually similar to HomeScreen but:
  - Hardcoded user name "Gurpreet"
  - Toggle state is local (not persisted to Firestore)
  - No `CircularProgressIndicator` rings on stat cards
  - Logout icon in AppBar (non-functional ‚Äî `onPressed: () {}`)
- This screen is shown after login via GoRouter redirect, but since GoRouter is not wired to MaterialApp, it is effectively unreachable in current state

### StepsScreen (`lib/screens/steps_screen.dart`)
- Summary card: hardcoded "4,880 total steps today"
- `fl_chart` `LineChart` with 6 hardcoded data points
- Chart color: `Colors.green`

### CaloriesScreen (`lib/screens/calories_screen.dart`)
- Summary card: hardcoded "480 kcal burned today"
- `fl_chart` `LineChart` with 6 hardcoded data points
- Chart color: `Colors.orange`

### ProfileScreen (`lib/screens/profile_screen.dart`)
- Reads `FirebaseAuth.instance.currentUser` for name + email
- Logout button: calls `FirebaseAuth.instance.signOut()` (triggers `AuthGate` stream ‚Üí redirects to `LoginScreen`)

## 10. Cloud Functions (Backend)

### checkPlantProteinPurchase (`functions/src/index.ts`)

**Type:** Firebase Callable Function (v2)
**Auth:** Required ‚Äî rejects unauthenticated calls with `unauthenticated` error
**Runtime:** Node.js 20

**Input:**
```json
{ "email": "user@example.com", "phone": "+1234567890" }
```
At least one of `email` or `phone` must be provided.

**Output:**
```json
{ "purchased": true }
```

**Flow:**
1. Validates Firebase Auth context
2. Validates input (email or phone required)
3. Calls `hasPurchasedPlantProtein()` from `shopify.ts`
4. Returns `{ purchased: boolean }`

**Secrets:**
- `SHOPIFY_ACCESS_TOKEN` ‚Äî via Firebase Secret Manager in production
- `SHOPIFY_SHOP_DOMAIN` ‚Äî via `defineString` / env var

### shopify.ts ‚Äî Shopify Admin GraphQL Client

**Shopify API Version:** `2024-01`
**Product checked:** `"Plant Protein"` (case-insensitive substring match on line item titles)

**Functions:**
| Function | Description |
|----------|-------------|
| `shopifyGraphQL<T>()` | Generic GraphQL request with auth headers |
| `fetchOrdersByEmail()` | Paginates orders filtered by `email:"x"` |
| `fetchOrdersByPhone()` | Looks up customer by phone, then fetches their orders |
| `hasPlantProteinInOrder()` | Checks if any line item title contains "plant protein" |
| `hasPurchasedPlantProtein()` | Main entry: tries email first, falls back to phone |

**Pagination:** Both order queries paginate with `first: 50` + `after: cursor` until `hasNextPage = false`.

**Phone normalization:** Strips all non-digits before querying.
