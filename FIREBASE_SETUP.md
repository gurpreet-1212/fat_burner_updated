# Firebase Integration Guide – Step by Step

This guide covers **Firebase Auth**, **Cloud Firestore**, and **Firebase Messaging (FCM)** in your Flutter app.

---

## Part 1: Firebase Console Setup

### Step 1.1 – Create a Firebase project

1. Go to [Firebase Console](https://console.firebase.google.com/)
2. Click **Add project**
3. Enter project name (e.g. `fat-burner`)
4. Enable or disable Google Analytics
5. Click **Create project**

### Step 1.2 – Enable services

1. In the left menu, go to **Build**:
   - **Authentication** → Get started → Enable **Email/Password** (and others as needed)
   - **Firestore Database** → Create database → Start in **test mode** (lock rules later)
   - **Cloud Messaging** – no extra setup; enabled by default

2. Note your **Project ID** in Project Settings (gear icon)

## Part 2: Install Tools

### Step 2.1 – Firebase CLI

```bash
# Install (or update) Firebase CLI
npm install -g firebase-tools

# Log in
firebase login
```

### Step 2.2 – FlutterFire CLI

```bash
dart pub global activate flutterfire_cli
```

Ensure the Dart global bin is on your PATH (often `%LOCALAPPDATA%\Pub\Cache\bin` on Windows).

## Part 3: Configure Flutter App

### Step 3.1 – Run FlutterFire Configure

From the **project root** (`App_Fat_Burner`):

```bash
dart run flutterfire_cli:flutterfire configure
```

Or:

```bash
flutterfire configure
```

This will:

- Ask you to select or create a Firebase project
- Create Android and iOS apps in Firebase (or match existing ones)
- Create `lib/firebase_options.dart` with config
- Add `android/app/google-services.json`
- Add `ios/Runner/GoogleService-Info.plist`
- Update Android Gradle with the `google-services` plugin

### Step 3.2 – Verify generated files

Confirm these exist:

- `lib/firebase_options.dart`
- `android/app/google-services.json`
- `ios/Runner/GoogleService-Info.plist`

## Part 4: Android Configuration

### Step 4.1 – Gradle (handled by FlutterFire)

If you used `flutterfire configure`, the Gradle setup is usually done. Otherwise:

1. **android/settings.gradle.kts** – add Google Services plugin:

```kotlin
plugins {
    id("dev.flutter.flutter-plugin-loader") version "1.0.0"
    id("com.android.application") version "8.11.1" apply false
    id("org.jetbrains.kotlin.android") version "2.2.20" apply false
    id("com.google.gms.google-services") version "4.4.2" apply false
}
```

2. **android/app/build.gradle.kts** – apply the plugin:

```kotlin
plugins {
    id("com.android.application")
    id("kotlin-android")
    id("dev.flutter.flutter-gradle-plugin")
    id("com.google.gms.google-services")  // Add this line
}
```

### Step 4.2 – minSdkVersion

Firebase needs `minSdk` at least **21** (often **23** recommended). In **android/app/build.gradle.kts**:

```kotlin
defaultConfig {
    applicationId = "com.fatburner.fat_burner"
    minSdk = 23
    // ...
}
```

### Step 4.3 – FCM on Android 8+

For notification channels, add to **android/app/src/main/AndroidManifest.xml** inside `<application>`:

```xml
<meta-data
    android:name="com.google.firebase.messaging.default_notification_channel_id"
    android:value="high_importance_channel" />
```

## Part 5: iOS Configuration

### Step 5.1 – Firebase

`flutterfire configure` adds `GoogleService-Info.plist`. Make sure it is referenced in Xcode under Runner.

### Step 5.2 – Push notifications and FCM

1. Open `ios/Runner.xcworkspace` in Xcode
2. Select **Runner** → **Signing & Capabilities**
3. Click **+ Capability**:
   - **Push Notifications**
   - **Background Modes** → enable **Remote notifications** and **Background fetch**

### Step 5.3 – APNs key (needed for FCM on iOS)

1. Go to [Apple Developer](https://developer.apple.com/account/) → **Certificates, Identifiers & Profiles** → **Keys**
2. Create a new key with **Apple Push Notifications service (APNs)** enabled
3. Download the `.p8` file and note Key ID and Team ID
4. In Firebase Console → **Project Settings** → **Cloud Messaging** → **Apple app configuration** → upload the APNs key and enter Key ID and Team ID

## Part 6: Initialization Code

Firebase is already wired in your project. The structure:

**lib/main.dart**
```dart
void main() async {
  WidgetsFlutterBinding.ensureInitialized();

  await Firebase.initializeApp(
    options: DefaultFirebaseOptions.currentPlatform,
  );

  FirebaseMessaging.onBackgroundMessage(firebaseMessagingBackgroundHandler);
  await FirebaseMessagingService.instance.initialize();

  runApp(const FatBurnerApp());
}
```

**Files:**
- `lib/main.dart` – Firebase and FCM init
- `lib/core/firebase_messaging_service.dart` – FCM permissions, token, foreground/background handlers
- `lib/firebase_options.dart` – Generated by `flutterfire configure` (placeholder until you run it)

**Important:** Run `flutterfire configure` before `flutter run`, or the app will throw because `firebase_options.dart` is not configured.

## Part 7: Firestore Security Rules (before production)

The app stores user profiles (UID, email, phone, displayName) in `users/{userId}`. Add rules in Firebase Console → **Firestore Database** → **Rules**:

```
rules_version = '2';
service cloud.firestore {
  match /databases/{database}/documents {
    match /users/{userId} {
      allow read, write: if request.auth != null && request.auth.uid == userId;
    }
  }
}
```

Adjust paths and rules for your data model.

## Troubleshooting

| Issue | Check |
|-------|--------|
| Android build fails | `google-services.json` in `android/app/`, plugin applied in `build.gradle.kts` |
| iOS build fails | `GoogleService-Info.plist` in Runner target, capabilities enabled |
| FCM token null on iOS | APNs key uploaded in Firebase Console, push capability enabled |
| Firestore permission denied | Update Firestore rules for your collections |
